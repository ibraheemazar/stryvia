# @stryvia/types

Shared TypeScript types used across the STRYVIA frontends and tooling — domain models (productions, scenes, brands, valuations), API request/response shapes, enums, and shared utility types.

The intent is that these types stay aligned with the Python engine's Pydantic schemas; a generation step (planned) will keep the two in sync.

**Currently a placeholder** — types are added as the engine API surface stabilises. See `docs/architecture/` for the staged build plan.
