# Scripts

Build, dev, and ops helper scripts that don't naturally belong to a single workspace — repo-wide bootstrap, environment setup, code generation (e.g. types from the engine schema), and cross-cutting maintenance utilities.

Scripts here are typically invoked via `npm run <name>` from the root, or directly during onboarding.

**Currently a placeholder** — helper scripts land as repeating manual steps emerge and become worth automating. See `docs/architecture/` for the staged build plan.
